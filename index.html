Jun 2025 — Nov 2025

Data Anonymization System for Financial Datasets

A pipeline that takes raw financial records and returns a dataset you can actually share — with identity-revealing fields protected and the underlying statistics still intact for downstream analysis.

  • Processes 10,000+ financial records, applying K-Anonymity (K=10) and L-Diversity (L=2) across 8+ sensitive fields, including Aadhaar numbers and bank account data.
  • Automates anonymization with deterministic SHA-256 hashing in Python and Pandas — cutting manual effort by 80% while keeping 90%+ statistical utility for analysis.
  • Ships as an interactive Streamlit dashboard with CSV upload, adjustable privacy levels, and export — bringing processing time down from minutes to under 30 seconds.
PythonPandasStreamlitScikit-LearnK-AnonymityL-Diversity
GitHub

The hardest part wasn't the hashing — it was deciding what "anonymous" actually means for a mixed dataset of names, account numbers, and behavioral fields. K-Anonymity groups records so no individual stands out from at least nine others with the same quasi-identifiers; L-Diversity then makes sure each group still has enough variation in sensitive values that grouping alone doesn't leak information. The Streamlit layer exists so a non-technical stakeholder can tune that privacy level themselves and see the utility trade-off before exporting.

Feb 2025 — Present

Zero Trust Secure API Gateway with Attack Detection

An API gateway that doesn't just log traffic — it learns what normal traffic looks like and flags the requests that don't fit, in real time.

  • Trains an Isolation Forest model on API traffic to detect SQL injection, XSS, and brute-force patterns, combining statistical scoring with rule-based filtering.
  • Visualizes traffic and threat data on an interactive dashboard, turning raw request logs into insights a non-technical stakeholder can actually use.
  • Structures backend request data with SQL to feed detection logic and support pattern analysis at query time.
PythonFastAPIScikit-LearnSQLIsolation Forest
GitHub

Isolation Forest works by isolating points through random splits — anomalies take fewer splits to isolate than normal traffic, which makes it cheap to score requests as they arrive instead of waiting on a labeled training set of "known attacks." I paired that statistical layer with rule-based checks for known SQL injection and XSS signatures, so the system catches both the attacks it's seen before and the ones that just look statistically wrong.
